# CLAUDE.md

Les 4 règles du CLAUDE.md personnel (Boris Cherny, créateur de Claude Code) — règles chargées à chaque session.

## 01 — Mode plan d'abord

**Écris le plan avant de coder.**

- Avant toute ligne de code, écris le plan complet.
- Si ça dérape en cours de route : STOP, refais le plan.
- Pas de code sans plan validé d'abord.

## 02 — Sous-agents pour le complexe

**Délègue pour garder le contexte propre.**

- Travail complexe = toujours un sous-agent.
- Garde le contexte principal léger et focus.
- 1 tâche complexe = 1 sous-agent dédié.

## 03 — Boucle d'auto-amélioration (game changer)

**Chaque erreur devient une règle.**

- Erreur détectée → transforme-la en règle.
- Sauvegarde-la dans le CLAUDE.md.
- Session suivante : -80% d'erreurs sur le même sujet.

## 04 — Prouve que ça marche

**Pas de "done" sans vérification.**

- Ne marque jamais une tâche terminée sans preuve.
- Exécute les tests + vérifie les logs à chaque fois.
- Pas de supposition : démontre que ça fonctionne.
